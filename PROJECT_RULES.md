# PROJECT_RULES.md — VideoRUBY

Краткие единые правила работы для всех помощников (Codex/Claude/Antigravity и др.)

## 1) Обязательные принципы (кратко)
- Язык общения: русский; код/идентификаторы/комментарии в коде — английский.
- Не логировать и не вставлять ключи/секреты в репозиторий.
- При сомнениях — задавать вопрос. При риске потери данных — остановиться.
- Любые изменения фиксировать в DEVLOG.md.
- Перед изменениями уточнять цель пользователя и ожидаемый результат.

## 2) Проектные соглашения
- FCPXML должен совпадать с параметрами исходного видео (fps/разрешение/длительность) и использовать frame‑aligned тайминги.
- Пути к файлам в XML — только абсолютные `file://` URL.
- GUI должен быть простым и понятным: минимум кликов, понятные подписи.

## 3) Безопасность
- Ключи хранить в Keychain; .env генерировать локально.
- Не коммитить .env, Credentials.env и любые секреты.

## 4) Формат обновлений
- Каждое изменение = запись в DEVLOG.md (дата, агент, что сделано, что дальше).
- Если есть спорные моменты — фиксировать альтернативы и причину выбора.
